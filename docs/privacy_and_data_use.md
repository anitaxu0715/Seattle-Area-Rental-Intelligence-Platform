# Privacy and Data Use

## Data Sources

All public enrichment data comes from official government open data portals:
- [City of Seattle Open Data Portal](https://data.seattle.gov/)
- [Seattle GeoData](https://data-seattlecitygis.opendata.arcgis.com/)

No data is scraped from commercial platforms (Zillow, Apartments.com, Google Maps, or apartment websites).

## Candidate Apartment Data

Candidate apartment data is manually curated from the project owner's apartment search notes. It is not extracted from any commercial listing platform.

The project supports two types of candidate files:

- **`candidate_apartments.example.csv`** — Public demo data with fictional addresses. Safe to commit and share. Included in the repository so the project is runnable by anyone.

- **`candidate_apartments.local.csv`** — Real apartment search notes stored locally. Gitignored and never committed.

## Privacy Safeguards

- `.gitignore` excludes `candidate_apartments.local.csv` from version control
- The `data_privacy_level` field tracks whether data is demo, portfolio-safe, or private
- Validation warns if placeholder patterns are detected in local data
- The loader logs which file is being used so the user always knows what data is active

## Risk Flag Language

Risk flags generated by this platform are **due-diligence signals, not legal or safety conclusions**.

Examples:
- `flag_registration_not_found` means the address was not found in the current rental registration dataset sample — not that the property is unregistered or illegal
- `flag_recent_complaints_nearby` means code violation records exist near the address — not that the property or neighborhood is unsafe
- `flag_active_construction_nearby` means active building permits exist nearby — not that the area is unsuitable

## Data Coverage Limitations

Seattle Open Data covers the City of Seattle. Apartments in neighboring cities (Shoreline, Bellevue, Renton, etc.) are not covered by Seattle city datasets.

The platform tracks this with jurisdiction-aware fields:
- `jurisdiction` identifies whether a candidate is in Seattle, Shoreline, or elsewhere
- `public_data_coverage_level` describes what public data is available for that jurisdiction
- `flag_partial_public_data_coverage` signals when a candidate is outside Seattle's data coverage — this is a **coverage limitation**, not a property risk

Shoreline candidates are valid search candidates. Missing Seattle records for Shoreline apartments should not be interpreted as negative evidence about the property. Shoreline and King County public records may need separate manual review or future data integration.

## Responsible Use

This platform is designed for personal due-diligence research. It is not a substitute for:
- Professional property inspections
- Legal advice about rental agreements
- Official city records searches
- Licensed real estate guidance
